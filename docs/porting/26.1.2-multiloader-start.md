# Better Lore 26.1.2 multiloader start

This snapshot starts the Fabric / NeoForge / Forge split without changing the current Fabric user experience.

## Current target

- Minecraft: 26.1.2
- Loaders in scope: Fabric, NeoForge, Forge
- Loaders out of scope: Quilt
- Versions out of scope for now: 1.20.1-1.20.4

## First technical split completed

### Common init

`AnvilLoreMod` no longer implements Fabric's `ModInitializer`. It is now a common bootstrap class with:

```java
AnvilLoreMod.init();
```

Fabric has loader-specific entrypoints:

```text
com.reign.betterlore.fabric.BetterLoreFabricMod
com.reign.betterlore.fabric.BetterLoreFabricClientMod
```

Forge and NeoForge now have equivalent tiny entrypoint drafts in loader source-set slices:

```text
src/forge/java/com/reign/betterlore/forge/BetterLoreForgeMod.java
src/neoforge/java/com/reign/betterlore/neoforge/BetterLoreNeoForgeMod.java
```

### Common networking facade

Mixin and UI code no longer call Fabric networking APIs directly.

Server/common code now goes through:

```text
com.reign.betterlore.net.AnvilLoreNetworking
com.reign.betterlore.net.BetterLoreNetworkingPlatform
```

Client code now goes through:

```text
com.reign.betterlore.client.net.ClientAnvilLoreNetworking
com.reign.betterlore.client.net.BetterLoreClientNetworkingPlatform
```

Fabric implementations are isolated in:

```text
com.reign.betterlore.net.fabric.FabricBetterLoreNetworkingPlatform
com.reign.betterlore.client.net.fabric.FabricBetterLoreClientNetworkingPlatform
```

NeoForge and Forge now have draft implementations of the same service interfaces in `src/neoforge/java` and `src/forge/java`. They must be compiled inside real loader modules before publishing.

### QuickText facade

Common formatting no longer directly depends on Text Placeholder API classes.

The common engine now loads optional real QuickText support through:

```text
com.reign.betterlore.lore.quicktext.QuickTextParserAdapter
```

Fabric provides:

```text
com.reign.betterlore.lore.quicktext.fabric.FabricPlaceholderApiQuickTextAdapter
```

If no adapter is available, Better Lore uses its clean-room QuickText-compatible fallback parser. This is the path intended for initial Forge/NeoForge ports if no stable Text Placeholder API port exists for the target loader.

6. Add Forge/NeoForge JEI/REI API dependencies and verify exclusion rectangles.
