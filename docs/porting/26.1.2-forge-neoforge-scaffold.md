# Better Lore 26.1.2 Forge/NeoForge scaffold

This snapshot moves the 26.1.2 port closer to real Forge and NeoForge builds without changing the working Fabric artifact yet.

## Added source-set slices

The project now contains loader-specific source slices that are intentionally not compiled by the current Fabric-only `build.gradle` yet:

- `src/neoforge/java`
- `src/neoforge/resources`
- `src/forge/java`
- `src/forge/resources`

These slices are meant to become the `neoforge` and `forge` modules when the Stonecutter + MultiLoader build layout is introduced.

## What is now loader-specific

The following runtime glue has Forge/NeoForge drafts:

- mod entrypoints
- serverbound/clientbound networking bridges
- client packet sending bridges
- loader metadata files
- ServiceLoader platform registration files

The shared code still owns the actual Better Lore behavior:

- anvil result/cost logic
- item name/lore storage
- QuickText-compatible fallback parser
- UI widgets
- JEI/REI exclusion bounds reported by the screen bridge

## Networking notes

NeoForge uses vanilla `CustomPacketPayload` codecs through `RegisterPayloadHandlersEvent` and `RegisterClientPayloadHandlersEvent`.

Forge uses a `SimpleChannel` draft. The channel is declared optional so vanilla/unmodded clients can still join a Better Lore server. Before publishing a Forge jar, verify the exact 26.1.2 Forge network API overloads in a real Forge workspace. In particular, the clientbound message registration overload may need to be adjusted to the exact Forge 26.1.2 `SimpleChannel` signature.

## QuickText notes

Fabric still uses Text Placeholder API when present. Forge/NeoForge currently use the clean-room Better Lore fallback parser, which is based on public QuickText behavior and public color math rather than copied source.

## Must pass before publishing Forge/NeoForge files

- Forge and NeoForge modules compile independently.
- Dedicated Forge/NeoForge servers start without client classes loading.
- Modded client + modded server works.
- Vanilla client + modded server can join.
- Modded client + vanilla server hides the custom UI.
- Name/lore edits, gradients, formatting, color picker, slider input, color wheel, and cursor insertion still work.
- JEI and REI do not overlap the Lore button, expanded panel, or color wheel.
- Old custom synced component registries do not return.
